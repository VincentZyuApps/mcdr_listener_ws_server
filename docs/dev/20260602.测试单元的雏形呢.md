# 测试单元雏形

## 事件流分析

### 服务端 → WS 客户端（出站）

| 事件 | 触发方式 | WS 消息格式 |
|------|---------|------------|
| 玩家进服 | MCDR 调用 `on_player_joined(server, player, info)` | `{"type": "player_join", "player_name": "xxx"}` |
| 玩家退服 | MCDR 调用 `on_player_left(server, player)` | `{"type": "player_leave", "player_name": "xxx"}` |
| 玩家聊天 | MCDR 调用 `on_user_info(server, info)` | `{"type": "player_chat", "player_name": "xxx", "content": "xxx"}` |

### WS 客户端 → 服务端（入站）

| 事件 | WS 消息格式 | 服务端动作 |
|------|------------|-----------|
| 文字消息 | `{"type": "chat_platform_to_server", ...}` | `server.say()` |
| 图片消息 | `{"type": "chat_platform_to_server", "images": [...]}` | `server.execute(tellraw)` |
| 远程命令 | `{"type": "external_command_to_server", ...}` | `server.rcon_query()` → 返回 `command_result` |

## 核心思路

不需要真正的 Minecraft 服务器。写一个 `tests/test_ws.py` 脚本，Mock 掉 MCDR 的 `PluginServerInterface`，启动真实的 `WebSocketHandler`，通过交互式命令行模拟各种事件。

```
┌─────────────────────────────────────────────────┐
│                  test_ws.py                      │
│                                                  │
│  1. Mock 一个 PluginServerInterface              │
│     - server.say() → print 到控制台              │
│     - server.execute() → print 到控制台          │
│     - server.rcon_query() → 返回模拟结果         │
│     - server.schedule_task() → 真正执行异步任务   │
│                                                  │
│  2. 启动真实的 WebSocketHandler                  │
│     - 监听 ws://0.0.0.0:60601                   │
│                                                  │
│  3. 提供一个交互式命令行菜单 + WS 客户端输出      │
└─────────────────────────────────────────────────┘
```

## Mock 接口映射

| MCDR 接口 | Mock 行为 |
|-----------|----------|
| `server.say(msg)` | 打印 `[SAY] msg` |
| `server.execute(cmd)` | 打印 `[EXEC] cmd` |
| `server.rcon_query(cmd)` | 返回 `"模拟RCON结果: {cmd}"` |
| `server.logger.info/warning/error` | 打印到控制台 |
| `server.is_on_async_executor_thread()` | 返回 `True` |
| `server.schedule_task(coro)` | 用 `asyncio.ensure_future()` 真正执行 |
| `server.create_task(coro)` | 同上 |
| `Info` 对象 | 用 `MagicMock` 构造，设置 `is_user=True`, `player=xxx`, `content=xxx` |

## 交互式命令设计

```
模拟服务端事件（出站测试）：
  j <player>     模拟玩家进服，WS 客户端应收到 player_join
  l <player>     模拟玩家退服，WS 客户端应收到 player_leave
  c <player> <msg>  模拟玩家聊天，WS 客户端应收到 player_chat

模拟客户端消息（入站测试）：
  t <msg>        发送文字消息，服务端应调用 server.say()
  i <url>        发送图片消息，服务端应调用 server.execute(tellraw)
  e <command>    发送远程命令，WS 客户端应收到 command_result

其他：
  q              退出
  h              帮助
```

## 测试场景清单

| # | 场景 | 操作 | 预期 |
|---|------|------|------|
| 1 | 玩家进服 | 输入 `j Steve` | WS 客户端收到 `{"type": "player_join", "player_name": "Steve"}` |
| 2 | 玩家退服 | 输入 `l Steve` | WS 客户端收到 `{"type": "player_leave", "player_name": "Steve"}` |
| 3 | 玩家聊天 | 输入 `c Steve hello` | WS 客户端收到 `{"type": "player_chat", ...}` |
| 4 | 文字消息转发 | 输入 `t 你好` | 控制台打印 `[SAY] [群名] 昵称: 你好` |
| 5 | 图片消息转发 | 输入 `i https://xxx.png` | 控制台打印 tellraw 命令 |
| 6 | 远程命令 | 输入 `e list` | WS 客户端收到 `{"type": "command_result", "ok": true, ...}` |
| 7 | Token 认证 | 用错误 token 连接 | 连接被拒绝 |
| 8 | 命令白名单 | 发送不在白名单的命令 | 返回 `ok: false` |
| 9 | 命令超时 | 模拟超时场景 | 返回 `ok: false, "timeout"` |
