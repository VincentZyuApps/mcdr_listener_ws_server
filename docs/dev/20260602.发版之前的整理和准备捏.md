# 📦 发版准备清单

> 对比 MCDR 官方模板 + 已发布插件（daycount-NBT / QuickBackupM / UnifiedHandler）分析出的差距

---

## 🔴 CRITICAL（必须修，否则无法发版）

### 1. 添加 `LICENSE` 文件
- **现状**：没有 LICENSE 文件
- **要求**：添加开源协议文件到仓库根目录（MIT / GPL-3.0 / BSD-3-Clause 等）
- **参考**：所有已发布插件都有 LICENSE

### 2. `resources` 字段包含 LICENSE
- **现状**：`"resources": ["lang", "resources"]`
- **要求**：改为 `"resources": ["lang", "resources", "LICENSE"]`
- **文件**：`mcdreforged.plugin.json`

### 3. `version` 格式修正
- **现状**：`"0.5.0-beta.1+20260602"`
- **要求**：去掉 `+20260602` build metadata，改为 `"0.5.0"` 或 `"0.5.0-beta.1"`
- **原因**：`+` 后缀会导致 MCDR 版本比较器异常，插件目录也无法正常识别

### 4. `link` 指向 GitHub
- **现状**：`"https://gitee.com/vincent-zyu/mcdr_listener_ws_server"`
- **要求**：改为 `"https://github.com/VincentZyuApps/mcdr_listener_ws_server"`
- **原因**：MCDR 插件目录基于 GitHub 索引

### 5. `description` 改为 i18n dict
- **现状**：单行中英文混合字符串
- **要求**：
```json
"description": {
    "en_us": "Group-server bridge beyond text: WebSocket server for MC, forwarding player chat/join/leave events to chat platforms; receiving text and images from platforms with in-game display entity rendering.",
    "zh_cn": "群服互通，不止于文字：在 MC 服务器开启 WebSocket 服务，将玩家聊天和进出服事件推送到聊天平台；接收平台文字与图片消息，图片以 display 实体在游戏内渲染展示。"
}
```

### 6. 创建 GitHub Actions release workflow
- **现状**：没有 `.github/workflows/`
- **要求**：创建 `.github/workflows/release.yml`，在 push `v*` tag 时自动：
  1. `python -m mcdreforged pack`
  2. 上传 `.mcdr` 到 GitHub Release
- **参考**：UnifiedHandler 的 `build.yml` 最完善

### 7. 注册 MCDR 插件目录
- **要求**：
  1. Fork `https://github.com/MCDReforged/PluginCatalogue`
  2. 添加 `plugins/mcdr_listener_ws_server.json`：
     ```json
     {
       "id": "mcdr_listener_ws_server",
       "authors": ["VincentZyu"],
       "repository": "https://github.com/VincentZyuApps/mcdr_listener_ws_server",
       "branch": "main",
       "related_path": ".",
       "labels": ["tool", "api"]
     }
     ```
  3. 提交 PR 到 PluginCatalogue

## 🟡 HIGH（强烈建议）

### 8. 添加 introduction 文件
- MCDR 插件目录会读取 `introduction.md` 和 `introduction-zh_cn.md` 作为介绍页
- 参考 daycount-NBT 的 `src/docs/introduction.md`
- 内容 1~3 段即可，概述插件功能

### 9. 移除 __init__.py 中的 counter 模板代码
- 当前 `__init__.py` 残留：
  ```python
  counter = old_module.counter + 1
  ```
- 这是 MCDR 模板的示例代码，发版前应清理

### 10. 清理 logs/ 运行时数据
- `logs/` 目录下有 `2025-*` 和 `2026-*` 的运行时 JSON 日志
- 执行：`git rm -r --cached logs/`
- 确保 `.gitignore` 中已有 `logs/`

### 11. 本地验证打包
- 执行：`python -m mcdreforged pack`
- 确认生成的 `.mcdr` 文件能正常安装到 MCDR 服务器

  git push github v0.5.0
